    LINQ to Objects

    Om modulet

    I dette model skal vi arbejde med LINQ (Language INtegrated Query), som er en af de mest geniale features i .NET. Det giver mulighed for at iterere, filtere, sortere og gruppere samlinger (lister, arrays med videre) af objekter i hukommelsen.

    Det er typisk svært for begyndere at forstå LINQ fordi det kræver en grundlæggende forståelse for blandt andet delegates og lambda, men du SKAL lære at bruge LINQ så du må tilbage i tidligere moduler hvis du falder af.

    Demo data

    At bruge FileInfo-objekter er en god måde at lege med LINQ på. Et FileInfo-objekt repræsenterer en fil på disken, og har en masse “brugbare” egenskaber:

    Det er nemt at oprette et array af FileInfo-objekter:

    // Skab en liste af filer i en mappe vi kan lege med
var mappe = new System.IO.DirectoryInfo(@"c:\temp");
var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
Console.WriteLine($"Har fundet {filer.Length} i {mappe.Name}");

    Sørg for at rette mappen og eventuelt søgestreng så du finder nogle filer at lege med.

    Derfor LINQ

    Du vil gerne slippe for at skrive kode som nedenstående hvis opgaven er at finde objekter i et array af filer hvor størrelsen er større end x bytes, og resultatet skal være sorteret efter navnet på filen:

    using System;
using System.Collections.Generic;
using System.IO;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Skab en liste af filer i en mappe vi kan lege med
            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
            Console.WriteLine($"Har fundet {filer.Length} i {mappe.Name}");

            var res1 = FindFilerStørreEndSorteretEfterNavn(filer, 100);
        }

        static List<FileInfo> FindFilerStørreEndSorteretEfterNavn(FileInfo[] filer, long størrelse)
        {
            List<FileInfo> tmp = new List<FileInfo>();
            foreach (var item in filer)
            {
                if (item.Length > størrelse)
                {
                    tmp.Add(item);
                }
            }
            tmp.Sort(new SorterFiler());
            return tmp;
        }
    }

    class SorterFiler : IComparer<FileInfo>
    {
        public int Compare(FileInfo x, FileInfo y)
        {
            return string.Compare(x.Name, y.Name);
        }
    }
}

    men i dette modul ser vi på det helt grundlæggende - LINQ to objects.

    LINQ to objects (System.Linq) arbejder med alle typer af objekter i C# - både objekter fra frameworket og egne objekter:

    • IEnumerable eller IEnumerable\<T> (generisk)
    • Array
    • List
    • Dictionary
    using System;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {

            // LINQ kan både arbejde med arrays, List, List<T> mv

            int[] array = { 3, 3, 5, 3, 2, 3, 5, 2 };
            var res1 = array.Where(i => i < 5).OrderBy(i => i);
            foreach (var item in res1)
                Console.WriteLine(item);

            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
            var res2 = filer.Where(i => i.Extension == ".zip").OrderBy(i => i.Name);
            foreach (var item in res2)
                Console.WriteLine(item.Name);

        }
    }
}

    LINQ to objects er en del af C# og virker ved hjælp af extension metoder - altså metoder der automatisk tilføjes klasser. Derfor er det nødvendigt at have en using-reference til System.Linq i de filer hvor du ønsker at bruge LINQ:

    using System.Linq;

    Syntaks

    Video Syntaks (4 min.)

    Se video

    Lidt forvirrende har LINQ to former for syntaks

    • “Query” syntaks
    • “Method” syntaks

    De fleste benytter “metode” (eller “lambda”) syntaksen men det er op til dig. Query-syntaksen minder lidt om SQL og kan derfor måske være nemmere at komme i gang med. Jeg vil dog anbefale dig at lære metode-syntaksen med det samme. Der kan dog være situationer hvor Query-syntaksen skaber nemmere læsbar kode, men jeg syntes du skal fokusere på metode-syntaksen.

    using System;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {

            // Skab en liste af filer i en mappe vi kan lege med
            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
            Console.WriteLine($"Har fundet {filer.Length} i {mappe.Name}");

            // finder alle filer med en størrelse over 1000 bytes sorteret efter navn

            // Query-syntaks (SQL)
            var res1 = from f in filer where f.Length > 1000 orderby f.Name select f;
            // Method-syntaks (Lambda)
            var res2 = filer.Where(i => i.Length > 1000).OrderBy(i => i.Name);

            Console.WriteLine($"Har fundet {res1.Count()} filer med Query-syntaks  - første fil er {res1.First().Name}");
            Console.WriteLine($"Har fundet {res2.Count()} filer med Method-syntaks - første fil er {res1.First().Name}");

        }
    }
}

    Defered/Imediate execution

    Video Defered/Imediate execution (4 min.)

    Se video

    Typisk vil resultatet af en query først ske når man anmoder om data og det kan ske når man eksempelvis itererer eller benytter ToList/ToArray metoden:

    using System;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {

            // LINQ kan både arbejde med arrays, List, List<T> mv

            int[] array = { 3, 3, 5, 3, 2, 3, 5, 2 };
            var res1 = array.Where(i => i < 5).OrderBy(i => i);

            // først her afvikles udtryk
            foreach (var item in res1)
                Console.WriteLine(item);


            // Nu afvikles udtryk med det samme
            var res2 = array.Where(i => i < 5).OrderBy(i => i).ToList();


        }
    }
}

    Brug af metoder i Lambda-syntaksen

    Video Brug af metoder i Lambda-syntaksen (10 min.)

    Se video

    Mange af LINQ metoderne tager som argument en Func (hvilket jo faktisk er en Predicate), og typisk skrives det som en ren lambda. Men blot for en god ordens skyld er der jo tale om en reference til en delegate.

    Her i en “rå” version:

    using System;
using System.IO;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Skab en liste af filer i en mappe vi kan lege med
            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
            Console.WriteLine($"Har fundet {filer.Length} i {mappe.Name}");

            bool findesFil(FileInfo f) {
                return f.Length > 1000;
            }
            Func<FileInfo, bool> f = findesFil;
            var res = filer.Where(f);

        }
    }
}

    Men det kan jo omskrives til lambda:

    using System;
using System.IO;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Skab en liste af filer i en mappe vi kan lege med
            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
            Console.WriteLine($"Har fundet {filer.Length} i {mappe.Name}");

            // Den komplete syntaks
            // Func<FileInfo, bool> f = (FileInfo i) => { return i.Length > 1000; };
            // som skrives nemmere som
            Func<FileInfo, bool> f = i => i.Length > 1000;

            var res = filer.Where(f);

        }
    }
}

    Og hvis man ikke skal bruge referencen f til noget kan man ligeså godt droppe variablen:

    using System;
using System.IO;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Skab en liste af filer i en mappe vi kan lege med
            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
            Console.WriteLine($"Har fundet {filer.Length} i {mappe.Name}");

            var res = filer.Where(i => i.Length > 1000);

        }
    }
}

    Sammenkædning

    Helt grundlæggende fungerer de fleste LINQ metoder ved at arbejde på en samling og returnere en ny. Man kan derfor vælge at sammenkæde kald

    using System;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {

            // Skab en liste af filer i en mappe vi kan lege med
            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
            Console.WriteLine($"Har fundet {filer.Length} i {mappe.Name}");

            var s1 = filer.Where(i => i.Length > 1000);
            var s2 = s1.OrderBy(i => i.Name);

            // samme som
            var s = filer.Where(i => i.Length > 1000).OrderBy(i => i.Name);

        }
    }
}

    Projecting (Select)

    Video Projecting (6 min.)

    Se video

    Select-funktionen kan bruges til at returnere en ny type. Her er der skabt en “lille” FileInfo-klasse fordi man måske ikke skal bruge andet end navn og længde:

    using System;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {

            // Skab en liste af filer i en mappe vi kan lege med
            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
            Console.WriteLine($"Har fundet {filer.Length} i {mappe.Name}");

            var res1 = filer.Where(i => i.Extension == ".cs").Select(i => new MinFileInfo { Navn = i.Name, Længde = i.Length }).ToList(); ;

            // nu er res1 en liste af MinFileInfo-objekter
            foreach (var item in res1)
                Console.WriteLine($"{item.Navn} - {item.Længde}");
        }
    }

    class MinFileInfo {
        public string Navn { get; set; }
        public long Længde { get; set; }
    }
}

    Anonyme typer

    Man man benytte anonyme typer til projektering i LINQ. På den måde behøver man ikke skabe en klasse/struktur. I virkelighed hører anonyme typer under den avancerede C#, men helt grundlæggende kan du skabe en anonym type med new {}, og benytter var ved typedefinition.

    var a = new { };
var c = new { navn = "a", id = 2 };

    Samme syntaxt kan bruges i LINQ:

    using System;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {

            // Skab en liste af filer i en mappe vi kan lege med
            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
            Console.WriteLine($"Har fundet {filer.Length} i {mappe.Name}");

            var res1 = filer.Where(i => i.Extension == ".cs").Select(i => new { Navn = i.Name, Længde = i.Length });

            // nu er res1 en liste af "anonyme" objekter uden en klasse definition
            foreach (var item in res1)
                Console.WriteLine($"{item.Navn} - {item.Længde}");
        }
    }
}

    Man kan sågar nøjes med at angive navnet på egenskaben hvis man ikke ønsker at ændre navnet:

    using System;
using System.Linq;

namespace Demo
{
    class Program

    Projektering i understrukturer (SelectMany)

    Man kan bruge SelectMany til at “flade” en struktur ud - altså arbejde med “børn” i stedet for “mødre”:

    class MinMappe {
    public string Navn { get; set; }
    public List<MinFil> Filer { get; set; }
}
class MinFil {
    public string Navn { get; set; }
}

    var mappe = new System.IO.DirectoryInfo(@"c:\temp");
var mineMapper = mappe.GetDirectories().Select(i => new MinMappe { Navn = i.FullName }).ToList();
foreach (var minMappe in mineMapper)
    minMappe.Filer = System.IO.Directory.GetFiles(minMappe.Navn).Select(i => new MinFil { Navn = i }).ToList();
// mineMapper består nu af en samling ala:
// c:\temp\mappe1
// -- fil1.txt
// -- fil2.txt
// c:\temp\mappe2
// -- fil3.txt
// ...

// (Select finder mapperne)
var res1 = mineMapper.Select(m => m.Navn).ToList();
// (SelectMany finder alle filerne)
var res2 = mineMapper.SelectMany(m => m.Filer).ToList();
// (SelectMany kan også tage mor med - her som et anonymt object)
var res3 = mineMapper.SelectMany(m => m.Filer, (m, f) => new { mappeName = m.Navn, filNavn = f.Navn }).ToList(); ;

    Gruppering (GroupBy)

    Video Gruppering (4 min.)

    Se video

    Gruppering er en af de effektive funktioner som LINQ tilbyder:

    using System;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Skab en liste af filer i en mappe vi kan lege med
            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
            Console.WriteLine($"Har fundet {filer.Length} i {mappe.Name}");

            // Grupperet efter extension
            var res1 = filer.GroupBy(i => i.Extension);

            foreach (var gruppe in res1)
            {
                // udskriver extension
                Console.WriteLine($"Gruppe: {gruppe.Key}");
                foreach (var fil in gruppe)
                {
                    // udskriver fil
                    Console.WriteLine($"\t{fil.Name}");
                }
            }
        }
    }
}

    Joining (Join)

    Video Join (4 min.)

    Se video

    Med Join kan man koble to (eller flere) lister sammen:

    using System;
using System.Collections.Generic;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Skab en liste af filer i en mappe vi kan lege med
            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
            Console.WriteLine($"Har fundet {filer.Length} i {mappe.Name}");

            List<MinGruppe> grupper = new List<MinGruppe>
            {
                new MinGruppe{ Extension=".txt", GruppeNavn = "Tekst filer" },
                new MinGruppe{ Extension = ".cs", GruppeNavn = "Kode filer" },
                new MinGruppe{ Extension = ".zip", GruppeNavn = "Komprimerede filer" },
                new MinGruppe{ Extension = ".csproj", GruppeNavn = "VS filer" },
                new MinGruppe{ Extension = ".sln", GruppeNavn = "VS filer" }
            };
            var mineFiler = filer.OrderBy(i=>i.Name).ToList();
            // nu er der to lister der kan "joines" på extension

            var res = mineFiler.Join(grupper, i => i.Extension, x => x.Extension, (i, x) => new { i.Name, x.GruppeNavn });
            foreach (var item in res)
                Console.WriteLine($"Fil: {item.Name} Gruppe: {item.GruppeNavn}");
        }
    }

    class MinGruppe
    {
        public string Extension { get; set; }
        public string GruppeNavn { get; set; }
    }
}

    Singleton

    Video Singleton (4 min.)

    Se video

    Der er tit brug for at finde enkelte elementer i en liste (eller resultat):

    using System;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Skab en liste af filer i en mappe vi kan lege med
            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
            Console.WriteLine($"Har fundet {filer.Length} i {mappe.Name}");

            // Find den første
            var item1 = filer[0];               // Finder den første eller fejler hvis liste er tom
            var item2 = filer.First();          // Finder den første eller fejler hvis liste er tom
            var item3 = filer.FirstOrDefault(); // Finder den første eller returnerer default værdi

            // vil være sikker på at der kun er en
            // var item4 = filer.Single();         // Finder den første - men der må kun være en eller en tom liste - ellers fejl
            // var item5 = filer.SingleOrDefault();// Finder den første - men der må kun være en eller en tom liste - ellers default værdi

            // Lidt mere avanceret (sørg for at minfil.txt er i /temp)
            var item6 = filer.FirstOrDefault(i => i.Name == "minfil.txt");  // Finder en fil
            var item7 = filer.FirstOrDefault(i => i.Name == "minfil1.txt"); // Finder ikke noget - derfor null
            var item8 = filer.Single(i => i.Name == "minfil.txt");          // Finder en fil (men havde der været to filer med det navn var det fejlet)

            // samme gælder med Last...
            var item9 = filer.LastOrDefault();

            // find "index 3"
            var item10 = filer.ElementAt(2);            // Finder index 2 (evt fejl hvis den ikke er der)
            var item11 = filer.ElementAtOrDefault(2);   // Finder index 2 (null ved fejl)

        }
    }

    Partitionering (Take/Skip)

    På Skip- og Take-funktioner til at “plukke” i en liste

    using System;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Skab en liste af filer i en mappe vi kan lege med
            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories);
            Console.WriteLine($"Har fundet {filer.Length} i {mappe.Name}");

            var res1 = filer.Take(10);          // første 10 filer
            var res2 = filer.Skip(10);          // Skip de første 10 filer og returner resten
            var res3 = filer.Skip(10).Take(10); // Skip de første 10 og returnerer de næste 10

        }
    }
}

    Sæt operationer

    Hvis man har to lister kan man bruge sæt-operationer til at sammenligne listerne.

    using System;
using System.Linq;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            // Skab lister af filnavne fra hver sin mappe
            var mappe = new System.IO.DirectoryInfo(@"c:\temp");
            var filer1 = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories).Select(i=>i.Name.ToLower()).ToList();
            Console.WriteLine($"Har fundet {filer1.Count} i {mappe.Name}");
            mappe = new System.IO.DirectoryInfo(@"c:\script");
            var filer2 = mappe.GetFiles("*.*", System.IO.SearchOption.AllDirectories).Select(i => i.Name.ToLower()).ToList();
            Console.WriteLine($"Har fundet {filer2.Count} i {mappe.Name}");

            var res1 = filer1.Except(filer2).ToList();   // Filer i filer1 som ikke er i filer2
            var res2 = filer2.Except(filer1).ToList();   // Filer i filer2 som ikke er i filer1
            var res3 = filer1.Union(filer2).ToList();    // Filer fra begge mapper (uden dubletter)
            var res4 = filer1.Intersect(filer2).ToList();// Filer der findes i begge mapper
            var res5 = filer1.Distinct().ToList();       // Fjern dubletter fra filer1
            var res6 = filer2.Distinct().ToList();       // Fjern dubletter fra filer2

            // concat kan bruges til at lægge samlinger sammen

        }
    }
}

    Udvidelse af LINQ

    Man kan have behov for at oprette egne LINQ udvidelser, og det kan nemt lade sig gøre med extension metoder
