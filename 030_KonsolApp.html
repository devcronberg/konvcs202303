    En konsol applikation

    Om modulet

    På kurset benytter vi udelukkende en C# konsol applikation så det er vigtigt du ved hvordan den er skruet sammen.

    En (.NET Core) konsol applikation

    Video Sådan opretter du en .NET Core Console app (4 min.)

    Se video

    En konsol-applikation er blot en ud af mange forskellige projekttyper, men vi arbejder med konsol-applikationer for at kunne sætte fokus på kode og ikke på UI. Se information om oprettelse af en konsol applikation i forrige modul.

    Skabelonen ser således ud:

    using System;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello world!");
        }
    }
}

    Namespace svarer til navnet på applikationen (men kan ændres) - klassen Program og metoden Main skal du ikke tage dig af lige nu. Det er blot skabelonen som runtime forventer, og vil automatisk afvikle kode i Main()-metoden.

    C# 9+ Konsol applikation

    Blot til orientering så kan man fra C# 9 skære en del af koden i skabelonen væk og blot nøjes med

    using System;

Console.WriteLine("Hello world!");

    I C# 10 kan man sågår også droppe “using System” som er implicit.

    For bedre at kunne forstå C# holder matarialet dog fast i brugen af den store skabelon:

    using System;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello world!");
        }
    }
}

    Du kan bruge ovennævnte som skabelon.

    Du kan også installere en extension i VS (vsix fil) med min .NET 6 skabelon. Find den på GitHub. Den kan installeres ved at hente og afvikle vsix-filen (luk VS inden installation). Herefter kan du finde skabelonen ved at søge på ‘classic’.

    Afvikling af en konsol applikation

    Forudsat at et konsol applikation projekt kan kompilere uden fejl er der som tidligere nævnt flere måder at afvikle koden på.

    • Byg og afvikling fra kommandolinjen
    • F6
    • ctrl+shift+b
    • Afvikling med debugger
    • F5
    • Afvikling uden debugger
    • ctrl+F5

    Man vil typisk altid afvikle med debuggeren for at få så meget hjælp som muligt.

    Helt grundlæggende om strenge

    En streng er en samling af tegn og findes i alle programmeringssprog. Der kommer meget senere mere om strenge senere men du vil falde over typen hele tiden så for en god ordens skyld er her en kort introduktion.

    En streng (datatypen hedder en string)i C# er omkranset af dobbeltplinger (“”) og kan sammenlægges med + operatoren:

    string a = "Dette er tekst";

string fornavn = "Mathias";
string efternavn = "Cronberg";
string navn = fornavn + " " + efternavn;    // Mathias Cronberg

    Som du vil se senere er string-typen ret omfattende og indeholder en masse metoder (gør til store/små bogstaver, fjern mellemrum, tæl antal tegn med videre).

    Yderligere vil du se brugen af string-templates i en masse eksempler - her og i eksempler på nettet. Det er en simpel og effektiv måde at sammenlægge strenge. Det sker ved at danne en skabelon med $”” hvor udtryk og variabler kan indsættes med {}.

    string fornavn = "Mathias";
string efternavn = "Cronberg";
string navn = $"Mit navn er {fornavn} {efternavn}"; // Mit navn er Mathias Cronberg

    Der er meget mere du skal vide om strenge - dette er blot en introduktion.

    Helt grundlæggende om metoder

    Video Helt grundlæggende om metoder (4 min.)

    Se video

    Inden vi kigger på udvikling af egne metoder er det vigtigt at du har en grundlæggende forståelse for brug af metoder. Alle typer har en eller flere metoder - nogle kan være relateret til typens (eventuelle) data, og andre er blot hjælpemetoder.

    Kald til metoder sker altid ved at angive metodens navn og angive argumenter i parentes. Nogle metoder returnerer en værdi - andre gør ikke.

    using System;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {

            Console.WriteLine();
            Console.WriteLine("Test");
            int res = Math.Max(10, 20);

        }
    }
}

    using

    Video using (4 min.)

    Se video

    En C# fil vil typisk altid i toppen have placeret nogle “using” instruktioner. Det er en måde at fortælle kompileren at den også skal søge efter typer i konkrete namespaces. Således kan følgende:

    namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            System.Console.WriteLine("Test");
            System.Random rnd = new System.Random();
            string data = System.IO.File.ReadAllText(@"c:\temp\data.txt");
            Cronberg.Hund hund = new Cronberg.Hund();
        }
    }
}

namespace Cronberg {
    class Hund { }
}

    skrives som

    using Cronberg;
using System;
using System.IO;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Test");
            Random rnd = new System.Random();
            string data = File.ReadAllText(@"c:\temp\data.txt");
            Hund hund = new Hund();
        }
    }
}

namespace Cronberg {
    class Hund { }
}

    Man sparer altså lidt referencer og gør koden nemmere og hurtigere at skrive. Kodeordet using står typisk i toppen af en fil, men kan også placeres i et namespace.

    Implicit using

    Fra C# 10 kan man i projekt filen aktivere implicit using instruktioner:

    <Project Sdk="Microsoft.NET.Sdk">

    <PropertyGroup>
        <OutputType>Exe</OutputType>
        <TargetFramework>net6.0</TargetFramework>
        <ImplicitUsings>enable</ImplicitUsings>
    </PropertyGroup>

</Project>

    Det vil automatisk sætte følgende globale usings (her en konsol app):

    using System;
using System.Collections.Generic;
using System.IO;
using System.Linq;
using System.Net.Http;
using System.Threading;
using System.Threading.Tasks;

    Global using

    Fra C# 10 er det muligt at benytte globale using instruktioner - evt i en fil for sig selv:

    Her er eksempelvis indholdet i en fil kaldet globalUsings.cs

    global using System.Text.Json;

    using - alias

    Man kan nogle gange have en fordel ved at benytte et alias - især hvis der er risiko for navne sammenfald:

    using IO = System.IO;
using Cronberg;
namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            IO.FileInfo f1;
            FileInfo f2;
        }
    }
}

namespace Cronberg {
    class FileInfo { }
}

    static using

    Brug af static using giver hurtig adgang til statiske medlemmer:

    using static System.IO.File;
using static System.Console;
namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            var s = ReadAllLines(@"c:\temp\data.txt");
            WriteLine(s);
        }
    }
}

    Brug af Console-klassen

    Video Brug af Console-klassen (5 min.)

    Se video

    Console-klassen består af en masse statiske metoder (man behøver ikke skabe en instans for at bruge dem), men i virkeligheden er den vigtigste WriteLine(). Den skriver blot ud på konsolen. Den kan eventuelt kombineres med Write(), som også udskriver blot uden linjeskift:

    using System;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {

            Console.WriteLine("Skriver en tekst...");
            string a = "Skriver en anden tekst";
            Console.WriteLine(a);

            // Tom linje
            Console.WriteLine();

            // Skriver uden linjeskrift
            Console.Write("a");
            Console.Write("b");
            Console.Write("c");
            Console.WriteLine();

        }
    }
}

    Det kan også være praktisk at kunne læse fra consol og måske skifte farve:

    using System;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {

            // Læs fra linje
            string input = Console.ReadLine();

            ConsoleKeyInfo k = Console.ReadKey();
            if (k.Key == ConsoleKey.A) { }
            if (k.Key == ConsoleKey.Escape) { }

            // Farve
            Console.ForegroundColor = ConsoleColor.Red;
            Console.WriteLine("Rød");
            Console.ForegroundColor = ConsoleColor.Gray;

        }
    }
}

    Argumenter til afvikling

    Når en konsol applikation startes fra kommandoprompt kan eventuelle argumenter aflæses gennem args-arrayet - ved eksempelvis:

    dotnet demo.dll /a /b

    består args af

    [0] "/a"
[1] "/b"

    Pakker til at hjælpe med args

    Der findes masser af NuGet pakker til at hjælpe med at splitte args - eksempelvis little-sharps/args eller commandlineparser/commandline. Sidstnævnte er nok den største og er rimelig nem i brug - her er en simpel skabelon:

    using System;
using System.Collections.Generic;
using CommandLine;

// Kræver NuGet pakken CommandLineParser
namespace ConsoleApp1
{
    class Program
    {
        static void Main(string[] args)
        {
            Parser.Default.ParseArguments<Options>(args)
                .WithParsed(Run)
                .WithNotParsed(HandleParseError);
        }

        private static void Run(Options opts)
        {
            string json = System.Text.Json.JsonSerializer.Serialize(opts);
            Console.WriteLine("Start with: " + json);
        }

        private static void HandleParseError(IEnumerable<Error> errs)
        {
            if (errs.IsVersion())
            {
                Console.WriteLine("More info about version");
                return;
            }

            if (errs.IsHelp())
            {
                Console.WriteLine("More info about help");
                return;
            }

            // More code related to error handling
        }


    }

    class Options
    {

        [Option('a', "text for a", Required = false, HelpText = "Help text for a")]
        public bool A { get; set; }

        [Option('b', "text for b", Required = true, HelpText = "Help text for b")]
        public string B { get; set; }

        [Option('c', "text for c", Required = false, HelpText = "Help text for c")]
        public int C { get; set; }


    }
}

    Husk at du nemt kan teste start fra kommandolinjen fra fanen debug på egenskaberne på selve projektet.

    Async console applikation

    Hvis du skrive kode der benytter async og await kan du med fordel erstatte den almindelige konsol skabelon fra:

    namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
        }
    }
}

    til

    using System.Threading.Tasks;

namespace Demo
{
    class Program
    {
        static async Task Main(string[] args)
        {
            // nu er await mulig
            await Task.Run(() => { });
        }
    }
}
