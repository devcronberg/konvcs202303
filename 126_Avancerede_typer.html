    Om modulet

    Dette modul gennemgår nogle af de lidt mere avancerede features relateret til typer.

    Records (poster)

    I C# 9 blev en feature kaldet records introduceret og den blev udvidet i C# 10. Records giver mulighed for hurtigt og nemt at skabe en immutable værdibaseret- (strukt) eller referencebaseret (klasse) type. En record vil autogenerere en klasse eller struct med følgende medlemmer:

    • overskrivning af Equals, GetHashCode mv for nem sammenligning
    • overskrivning af == og != operatorerne
    • overskrivning af ToString
    • Evt automatisk generering af felter, egenskaber og konstruktør
    • En deconstruct metode til nedbrydning til variabler

    I sin simple syntaks ser en record ud som følger (her en class)

    public record class Person(string Navn, int Alder);

// class er default så følgende er det samme
// public record Person(string Navn, int Alder);

    Det gør det muligt at benytte klassen som en immutable datatype

    Person p1 = new Person("a", 11);

Console.WriteLine(p1);
// Person { Navn = a, Alder = 11 }

Person p2 = p1;
Console.WriteLine(p2);
// Person { Name = a, Alder = 11 }

Person p3 = p1 with
{
    Alder = 12
};
Console.WriteLine(p3);
// Person { Name = a, Alder = 12 }


Person p4 = new Person("b", 15);

// Sammenligner værdier!!
Console.WriteLine(p3==p4);  // false


// Deconstruct
(var n, var a) = p4;
Console.WriteLine(n);   // b
Console.WriteLine(a);   // 15

    samme er muligt hvis man ønsker en stuct i stedet for en klasse:

    public record struct Person(string Navn, int Alder);

    hvilket skabe samme medlemmen - men nu som en struct.

    Tilretning af record

    Den hurtige syntaks kan afløse af lidt mere kode men også mere styring - her er eksempelvis en default konstruktør

    public record class Person(string Navn, int Alder)
{

    public string? Land { get; init; }

    public void Skriv() {
        Console.WriteLine($"Jeg hedder {Navn}, er fra {Land}, og er {Alder} år gammel");
    }

}

    som kan bruges som

    Person person = new Person("a", 10) { Land = "DK" };
Console.WriteLine(person);
person.Skriv();

    Brug af with til kopi

    For nemt at skabe en ny instans kan with-kodeordet benyttes

    Person person1 = new Person("a", 10, "DK")
Person person2 = person1 with { Navn = "b" };
// person2 == person1 bortset fra navn

    Mulighed for at værdibaserede variabler kan få værdien null

    • Nullable\<T>
      • Bemærk at Nullable\<T> er en struct
    • Brug af ? ved erklæring
    • Brugt en del til at tilføje “uinitialiseret” til alle structs
    • Brugt en del i ORM produkter til at mappe en database tabel
    • Bemærk - C# 8 har også en nullable reference (string?), men det er noget helt andet!
    {
    Nullable<int> a = new Nullable<int>();
    a = 10;
    Console.WriteLine(a);                       // 10
    Console.WriteLine(a.HasValue);              // true
    Console.WriteLine(a.GetValueOrDefault());   // 10
    Console.WriteLine(a is null);               // false
    Console.WriteLine(a == null);               // false
    a = null;
    Console.WriteLine(a.HasValue);              // false
    Console.WriteLine(a.GetValueOrDefault());   // 0
    Console.WriteLine(a.GetValueOrDefault(20)); // 20
    Console.WriteLine(a is null);               // true
    Console.WriteLine(a == null);               // true
}

{
    int? a;
    a = 10;
    Console.WriteLine(a);                       // 10
    Console.WriteLine(a.HasValue);              // true
    Console.WriteLine(a.GetValueOrDefault());   // 10
    Console.WriteLine(a is null);               // false
    Console.WriteLine(a == null);               // false
    a = null;
    Console.WriteLine(a.HasValue);              // false
    Console.WriteLine(a.GetValueOrDefault());   // 0
    Console.WriteLine(a.GetValueOrDefault(20)); // 20
    Console.WriteLine(a is null);               // true
    Console.WriteLine(a == null);               // true
}

  public T GetValueOrDefault();
  public T GetValueOrDefault(T defaultValue);
  // ...
}

    Tuples

    Generisk datastruktur som især kan bruges til at returnere data fra en funktioner uden at angive en type, som argumeter eller som en midlertidig datatype.

    Der findes i version 8 to muligheder.

    • System.Tuple (class - immutable)
    • System.ValueTuple (struct - mutable)
    var a = (1, 1);
Console.WriteLine(a.GetType()); // System.ValueTuple
System.Tuple<int, int> b = new Tuple<int, int>(1, 1);
Console.WriteLine(b.GetType()); // System.Tuple
System.Tuple<int, int> c = Tuple.Create(1, 1);
Console.WriteLine(c.GetType()); // System.Tuple
var d = (nr1: 1, nr2: 2);
Console.WriteLine(d.GetType()); // System.ValueTuple
System.ValueTuple<int, int> e = ValueTuple.Create(1, 1);
Console.WriteLine(e);   // (1, 1)

// Immutable - Mutable
Tuple<int, int> aa = new Tuple<int, int>(2, 2);
// aa.Item1 = 1;
ValueTuple<int, int> bb = ValueTuple.Create(2, 2);
bb.Item1 = 1;

    Bruges som nævnt blandt i metodekald og returværdier:

    using System;
using System.Collections.Generic;
using System.Threading.Tasks;

namespace Demo
{
    class Program
    {

        static async Task Main(string[] args)
        {

            var res1 = Test1(("a", "b"));
            if (res1.Item2)
            {
                Console.WriteLine(res1.Item1);
            }


            //var res2 = Test2(("a", "b"));
            // Eller...
            var res2 = Test2(ValueTuple.Create<string, string>("a", "b"));
            if (res2.succes)
            {
                Console.WriteLine(res2.længde);
            }
        }

        static (int, bool) Test1((string, string) v)
        {
            if (string.IsNullOrEmpty(v.Item1) || string.IsNullOrEmpty(v.Item2))
                return (0, false);
            else
                return (v.Item1.Length + v.Item2.Length, true);
        }

        static (int længde, bool succes) Test2((string txt1, string txt2) v)
        {
            if (string.IsNullOrEmpty(v.txt1) || string.IsNullOrEmpty(v.txt1))
                return (0, false);
            else
                return (v.txt1.Length + v.txt2.Length, true);
        }
    }
}

    Nedbrydning

    Tuples ses tit i forbindelse nedbrydning til enkelte variabler:

    // Definering af Tuple (kunne også være returværdi fra metode)
var a = (navn: "Mikkel", alder: 16);
var (navn1, alder1) = a;
(string navn2, int alder2) = a;

// Kun alder eller navn
var (_, alder3) = a;
(string navn3, _) = a;

    Ved egne klasser kan du selv skabe en Deconstruct:

    class Person
{
    public string Navn { get; set; }
    public int Alder { get; set; }
    public bool ErDansk { get; set; }

    public void Deconstruct(out string navn,
        out int alder, out bool erDansk)
    {
        navn = Navn;
        alder = Alder;
        erDansk = ErDansk;
    }
}

    Metoden benytter out-argumenter som betyder, at der sendes en reference med til variablen og ikke selve værdien. Derfor kan metoden tildele argumenterne en værdi.

    Nu kan der foretages en nedbrydning:

    Person p = new Person
{
    Navn = "Mathias",
    Alder = 14,
    ErDansk = true
};
Console.WriteLine(p.Navn);      // Mathias
Console.WriteLine(p.Alder);     // 14
Console.WriteLine(p.ErDansk);   // true

// Nedbrydning
(var navn, var alder, var erDansk) = p;
Console.WriteLine(navn);        // Mathias
Console.WriteLine(alder);       // 14
Console.WriteLine(erDansk);     // true

    Metoden kan også overloades

    class Person
{
    public string Navn { get; set; }
    public int Alder { get; set; }
    public bool ErDansk { get; set; }

    public void Deconstruct(out string navn, out int alder,
        out bool erDansk)
    {
        navn = Navn;
        alder = Alder;
        erDansk = ErDansk;
    }

    public void Deconstruct(out string navn, out int alder)
    {
        navn = Navn;
        alder = Alder;
    }
}

    Discards

    Ved nedbrydning er der nogle gange ikke behov for variabler, og her kan discards (_) benyttes til at fortælle udvikler og kompiler at en værdi ignoreres:

    namespace Demo
{
    internal class Program
    {
        static void Main(string[] args)
        {

            Person person = new Person() { Navn = "a", Alder = 10, ErDansk = true };

            (var navn, var alder, var erDansk1) = person;
            (_, _, var erDansk2) = person;
            Console.WriteLine(erDansk2);

        }
    }

    class Person
    {
        public string? Navn { get; set; }
        public int Alder { get; set; }
        public bool ErDansk { get; set; }

        public void Deconstruct(out string? navn, out int alder,
            out bool erDansk)
        {
            navn = Navn;
            alder = Alder;
            erDansk = ErDansk;
        }
    }
}

    Mutable og Immutable data

    “In object-oriented and functional programming, an immutable (unchangeable) object is an object whose state cannot be modified after it is created. This is in contrast to a mutable (changeable) object, which can be modified after it is created.” WikiPedia

    // Hvorfor er value types immutable ;)
5++;    // PS - kan ikke kompilere
Console.WriteLine(5);

    Mange struct’s i frameworket er immutable

    DateTime d = new DateTime(2019, 1, 1);
Console.WriteLine(d);   // 2019-1-1
d.AddDays(1);
Console.WriteLine(d);   // 2019-1-1
d = d.AddDays(1);
Console.WriteLine(d);   // 2019-1-2

    men det er et spørgsmål om implementation:

    struct MutableStruct
{
    private int state;

    public MutableStruct(int state) { this.state = state; }

    public void ChangeState() { this.state++; }
}

struct ImmutableStruct
{
    private readonly int state;

    public ImmutableStruct(int state) { this.state = state; }

    public ImmutableStruct ChangeState()
    {
        return new ImmutableStruct(this.state + 1);
    }
}

    Det klassiske eksempel er System.String (immutable) og System.Text.StringBuilder (mutable).

    Immutable klasser

    Men kan relativt nemt skabe immutable klasser ved at indkapsle felter.

    using System;
using System.Threading.Tasks;

namespace Demo
{
    class Program
    {

        static async Task Main(string[] args)
        {

            PersonMutable a = new PersonMutable("a", 10);
            Console.WriteLine(a.Alder);
            Test(a);
            Console.WriteLine(a.Alder);

            PersonImutable b = new PersonImutable("a", 10);
            Console.WriteLine(b.Alder);
            Test(b);
            Console.WriteLine(b.Alder);

        }

        static void Test(PersonMutable p)
        {
            p.Alder = 100;
        }

        static void Test(PersonImutable p)
        {
            // p.Alder = 100; -- fejl
        }

    }

    class PersonMutable
    {
        public string Navn { get; set; }
        public int Alder { get; set; }

        public PersonMutable(string navn, int alder)
        {
            this.Navn = navn;
            this.Alder = alder;
        }
    }

    class PersonImutable
    {
        public string Navn { get; private set; }
        public int Alder { get; private set; }

        public PersonImutable(string navn, int alder)
        {
            this.Navn = navn;
            this.Alder = alder;
        }
    }
}

    DynamicObject

    • oprettelse af et objekt på runtime
    • masse metoder til overskrivning
    using System.Collections.Generic;
using System.Dynamic;

namespace Dynamic
{
    public class MitObjekt : DynamicObject
    {

        Dictionary<string, object> properties = new Dictionary<string, object>();

        public override bool TryGetMember(GetMemberBinder binder, out object result)
        {
            if (properties.ContainsKey(binder.Name))
            {
                result = properties[binder.Name];
                return true;
            }
            else
            {
                result = "Forkert egenskab";
                return false;
            }
        }

        public override bool TrySetMember(SetMemberBinder binder, object value)
        {
            properties[binder.Name] = value;
            return true;
        }

    }
}

    dynamic o = new MitObjekt();
o.navn = "x";
o.alder = 2;

 ---------- Output: ----------

    Anonyme typer

    • Anonyme typer giver mulighed for at skabe typer uden en skabelon (class/struct)
    • Benyttes meget ifbm med LINQ
    • Simple “klasseløse” datastrukturer
      • felter er readonly
      • er sealed
      • ingen metoder mv
    // tomt objekt
var a = new { };

// datastruktur
dynamic b = new { navn = "a", id = 2 };

// evt som argument (kræver dynamic)
Test(b);

// LINQ
var col = new List<Person>() {
    new Person() { Id = 1, Navn = "a", Alder = 10 },
    new Person() { Id = 2, Navn = "b", Alder = 11 }
};
var res = col.Where(i => i.Alder > 10).Select(i => new { n = i.Navn, i = i.Id });
foreach (var item in res)
    Console.WriteLine(item.i);


void Test(dynamic s)
{
    Console.WriteLine(s.id);    // ingen intellisense
    // Console.WriteLine(s.ip);    // fejl
}



/*
 ---------- Output: ----------

2
2

*/
