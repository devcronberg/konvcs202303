    Stukturer

    Om modulet

    Der er to former for skabeloner til at skabe typer i .NET. Klasser (class) er klart mest brugt, men man kan også benytte strukturer (struct). I grundlæggende C# er egne strukturer ikke så meget brugt, men i mere avanceret kode kan man benytte struct’s i blandt andet optimeringssammenhæng.

    Hvad er en struct

    En struct er ligesom en class en skabelon for en instans/objekt, men den placeres (i langt de fleste tilfælde) et specielt sted i hukommelsen, og en instans indeholder værdier (og ikke referencer). Typen er især brugt til små typer som benyttes meget, og som skal være nemme at kopiere og sammenligne.

    I den grundlæggende C# kan du se helt bort fra definering af dine egne strukturer, så bortset fra denne sektion kan du godt hoppe videre til næste modul.

    Indbyggede strukturer

    Der findes et hav af strukturer i framework som du benytter hele tiden - herunder alle de simple variabler som int, double, bool med videre. De er stukturer både af praktiske og performancemæssige årsager

    • Det er nemt at kopiere værdier
    • De “koster” ikke så meget for runetime ved oprettelse og (især) oprydning
    using System;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {
            // nemt og effektivt
            int i = 1;

            // giver det samme
            int j = new int();
            j = 1;

            bool k = true;
            DateTime l = DateTime.Now;
            double m = 0;
            // + mange andre
        }

    }
}

    Definering af egne strukturer

    En struktur kan defineres med struct-kodeordet (ligesom class), og kan i virkeligen består af det samme som klasser:

    • felter
    • metoder
    • egenskaber
    • hændelser

    Her er et eksempel på brug af en simpel stuktur til opbevaring af data:

    internal struct Person
{
    public string Navn;
    public int Alder;
}

    Den store forskel på strukturer og klasser er, at instanser af strukturer placeres et andet sted i hukommelsen og, at strukturer er værdibaserede (der gemmes værdier i variabler) medens klasser er referencebaserede (der gemmes referencer i variabler).

    Oprettelse af instanser af structs

    Man kan oprette instanser både med og uden new kodeordet, men bedst pratice er at benytte new. Det vil sikre, at felter bliver tildelt default værdier

    using System;

namespace Demo
{
    class Program
    {
        static void Main(string[] args)
        {

            int a;
            // a++;     // Fejl - a er ikke initialiseret
            a = 0;
            a++;        // ok - a = 1

            int b = new int();
            b++;        // ok - b = 1

            MinStruct c;
            // c.A++;   // Fejl - c er ikke initialiseret
            MinStruct d = new MinStruct();
            // d.A = 0
            // d.B = false
            // d.C = 01-01-0001 00:00:00

        }
    }

    struct MinStruct
    {
        public int A;
        public bool B;
        public DateTime C;
    }
}

    Her er en kort animieret gif - en gul linje viser at der skal til at blive afviklet! Du skal være helt skarp på hvad der sker:

    • En variabel på stack’en (del af hukommelse) indeholder værdierne af en struct
    • new-kodet er ikke nødvendigt (men best pratice)
    • Når en variabel sættes = en anden variabel kopieres værdi (ikke reference)

    Forskellen på en struct og en class

    • class er en referencebaseret type og placeret på heap’en
      • kan have en null-værdi
      • koster performance ved allokering og ved oprydning
      • supporterer nedarvning
      • kan implementere interfaces
    • struct er en værdibaseret type og (typisk) placeret på stack’en
      • har altid en værdi
      • supporterer ikke nedarvning
      • kan dog implementere interfaces
      • lidt anden implementation til brug af constructor

    Constructor

    Man kan benytte construktor som ved klasser, men man må ikke benytte en default constructor (uden argumenter), og alle felter skal initialiseres.

    using System;

namespace DebugDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Person p1 = new Person();           // default constructor kan ikke overskrives/fjernes
            Person p2 = new Person("a", 10);
        }

    }
}

internal struct Person
{
    public string Navn;
    public int Alder;

    public Person(string navn, int alder)
    {
        this.Navn = navn;
        this.Alder = alder;
    }
}

    Rekursive strukturer

    På grund af deres placering på stakken må stukturer ikke være rekursive.

    // ok
class Node {
    public object Data { get; set; }
    public Node Next { get; set; }
}

// ikke ok
struct Node {
    public object Data { get; set; }
    public Node Next { get; set; }
}

    Men klasser (og metoder) må jo naturligvis gerne.
