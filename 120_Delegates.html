    Delegates

    Om modulet

    I dette modul ser vi på delegates og lambda udtryk. Begge ting er nok noget af det begyndere har alle sværest ved i C#, og det skyldes egentlig ikke at teorien er svær men mere at syntaksen kan være lidt teknisk, og at der er mange måde at skrive det samme på. Men tungen lige i munden og start med at forstå hvad en delegate overhovedet er - så må du øve dig på syntaksen over lidt tid.

    Bare for en god ordens skyld - skulle du efter at have stiftet bekendtskab med delegates og lambda ikke helt have set lyset så er du ikke den første. Giv det er par dage og så se videoerne igen og lav opgaverne.

    Grundlæggende om delegates

    Video Grundlæggende om delegates (4 min.)

    Se video

    Vi har snakket om lokale og statiske reference til instanser på både heap og stack, men man kan også arbejde med referencer til metoder.

    Ideen med at kunne arbejde med reference til en eller flere metoder er typisk at afkoble implementation - udvikleren der ønsker at benytte vores klasse eller metode må sende en reference til kode han/hun ønsker afviklet i et givet situation.

    Hvad er en delegate

    En delegate er en speciel type i C# som kan ligge til grund for objekter der indeholder referencer til en eller flere metoder:

    Tænk på ordet “delegate” - en type der give mulighed for at “delegere” afvikling af metoder. Kan sammenlignes med en objektorienteret version af C++’s funktionspointere.

    Hvad kan man bruge en delegate til

    Video Hvad kan man bruge en delegate til (12 min.)

    Se video

    Man kunne forestille sig:

    • finde elementer i en liste eller array ud fra et boolsk (true/false) resultat af en metode
    • et argument til en metode hvor der angives en refence til metoder man ønsker afviklet ved success eller fejl (hedder også en call back metode og er meget benyttet i andre sprog)
    • når der oprettes en instans af en klasse skal der samtidigt angives hvilken/hvilke metoder man ønsker klassen skal bruge til log
    • en dynamisk “regnemaskine” hvor udvikleren selv kan udvide de funktioner lommeregneren stiller til rådighed
    • når terningen slår en sekser så bliver der afviklet en eller flere metoder angivet af udvikleren der opretter terningen (hedder også “hændelser”)

    Du kan finde koden der dækker ovennævnte eksempler her.

    Opret et delegate objekt

    Video Opret et delegate objekt (4 min.)

    Se video

    Når først en delegate er defineret kan der skabes objekter på flere måder

    • brug af new til at oprette objekt
    • constructor kræver en reference til en metode
    • brug af navnet på metoden
    • compiler autogenererer kode

    Husk at signatur på metoden skal matche signatur på delegate (ikke helt sandt i et arvehieraki men se bort fra det lige nu)

    using System;

namespace Demo
{
    // Defination af delegates
    public delegate void MinDelegate1();
    public delegate void MinDelegate2(string txt);
    public delegate int MinDelegate3(int a, int b);

    internal class Program
    {
        private static void Main(string[] args)
        {

            MinDelegate1 o1 = new MinDelegate1(Test1);
            MinDelegate1 o2 = Test1;

            MinDelegate2 o3 = new MinDelegate2(Test2);
            MinDelegate2 o4 = Test2;

            MinDelegate3 o5 = new MinDelegate3(Test3);
            MinDelegate3 o6 = Test3;

            // Man kan i princippet oprette et objekt med en NOP (no operation)
            MinDelegate1 o7 = new MinDelegate1(NOP);
            MinDelegate1 o8 = NOP;
            MinDelegate1 o9 = delegate
            { };


            // Metoder kan defineres lokalt eller som instans eller statiske metoder
            void NOP()
            { }
            void Test1()
            { }
            void Test2(string t)
            { }
            int Test3(int a, int b)
            { return 0; }

            if (System.Diagnostics.Debugger.IsAttached)
            {
                Console.Write("Press any key to continue . . . ");
                Console.ReadKey();
            }
        }

    }

}

    Flere referencer i en delegate

    Video Flere referencer i en delegate (6 min.)

    Se video

    En delegate består af en liste af metoder så man kan både tilføje og fjerne referencer dynamisk.

    • Alle bliver afviklet

    Brugen af += bliver erstatet af et kald til Combine().

    using System;

namespace Demo
{
    public delegate void MinDelegate1(string txt);

    internal class Program
    {
        private static void Main(string[] args)
        {

            //MinDelegate1 o1 = new MinDelegate1(SkrivPåKonsol);
            MinDelegate1 o1 = SkrivPåKonsol;
            // Ekstra metoder
            o1 += GemIFil;
            o1 += Console.WriteLine;

            // Afvikler SAMTLIGE (tre) metoder
            o1?.Invoke("Test");
            // eller
            // if(o1!=null)
            //   o1("Test");

            // Man kan også afvikle dem en af gangen (brugbar ved retur værdier)
            foreach (var item in o1.GetInvocationList())
                item.DynamicInvoke("NY TEST");

            void SkrivPåKonsol(string txt)
            {
                Console.WriteLine($"I SkrivPåKonsol med {txt} som argument");
            }

            void GemIFil(string txt)
            {
                System.IO.File.AppendAllText(@"c:\temp\test.txt", txt + "\r\n");
            }

            if (System.Diagnostics.Debugger.IsAttached)
            {
                Console.Write("Press any key to continue . . . ");
                Console.ReadKey();
            }
        }
    }
}

    Fjern referencer fra delegates

    Video Fjern referencer fra delegates (1 min.)

    Se video

    Man kan fjerne referencer med -= og bør gøre det. Når man har med events/referencer at gøre kan man komme til at skabe et memoryleak - så værd bevidst om at en reference bør fjernes igen.

    Brugen af -= bliver erstatet af et kald til Remove().

    using System;

namespace Demo
{
    public delegate void MinDelegate1(string txt);

    internal class Program
    {
        private static void Main(string[] args)
        {

            MinDelegate1 o1 = SkrivPåKonsol;
            // Ekstra metoder
            o1 += GemIFil;
            o1 += Console.WriteLine;

            o1?.Invoke("Test");

            o1 -= GemIFil;
            o1 -= Console.WriteLine;
            o1 -= SkrivPåKonsol;

            // eller
            // o1 = null;

            void SkrivPåKonsol(string txt)
            {
                Console.WriteLine($"I SkrivPåKonsol med {txt} som argument");
            }

            void GemIFil(string txt)
            {
                System.IO.File.AppendAllText(@"c:\temp\test.txt", txt + "\r\n");
            }

            if (System.Diagnostics.Debugger.IsAttached)
            {
                Console.Write("Press any key to continue . . . ");
                Console.ReadKey();
            }
        }
    }
}

    DynamicInvoke

    Man kan eventuelt benytte den (i argumetnerne) typesvage DynamicInvoke-metode. Den skal også benyttes hvis man skal kalder metoder fra GetInvocationList() med mindre man typekonverterer:

    Func<int, int, int> c = (a, b) => a + b;
c += (a, b) => a - b;
c += (a, b) => a * b;
c += (a, b) => a / b;
foreach (var item in c.GetInvocationList())
{
    int r = (int)item.DynamicInvoke(8, 2);
    Console.WriteLine(r);
}

    Action

    Video Action (4 min.)

    Se video

    Action dækker void metoder (med 16 argumenter), og typerne på argumenterne angives generisk ved erklæring. Således er følgende tre delegates erstattet af Action-delegates:

    using System;

namespace Demo
{
    public delegate void MinDelegate1();
    public delegate void MinDelegate2(string txt);
    public delegate void MinDelegate3(string txt, int a);

    internal class Program
    {
        private static void Main(string[] args)
        {

            MinDelegate1 f1 = Test1;
            Action f2 = Test1;

            MinDelegate2 f3 = Test2;
            Action<string> f4 = Test2;

            MinDelegate3 f5 = Test3;
            Action<string, int> f6 = Test3;


            f1?.Invoke();
            f2?.Invoke();
            f3?.Invoke("");
            f4?.Invoke("");
            f5?.Invoke("", 0);
            f6?.Invoke("", 0);

            void Test1()
            { Console.WriteLine("I Test1"); }
            void Test2(string t)
            { Console.WriteLine("I Test2"); }
            void Test3(string t, int x)
            { Console.WriteLine("I Test3"); }

            if (System.Diagnostics.Debugger.IsAttached)
            {
                Console.Write("Press any key to continue . . . ");
                Console.ReadKey();
            }
        }

    Func

    Video Func (4 min.)

    Se video

    Func dækker metoder med returværdi, og typen på argumenter og på returværdi angives generisk. Den sidst angivne type dækker returværdi - Eksempelvis Func<int, double, bool> d1 svarer til delegate bool d1(int a, double b)

    Således er følgende tre delegates erstattet af Func-delegates:

    using System;

namespace Demo
{
    public delegate bool MinDelegate1();
    public delegate bool MinDelegate2(string txt);
    public delegate bool MinDelegate3(string txt, int a);

    internal class Program
    {
        private static void Main(string[] args)
        {

            MinDelegate1 f1 = Test1;
            Func<bool> f2 = Test1;

            MinDelegate2 f3 = Test2;
            Func<string, bool> f4 = Test2;

            MinDelegate3 f5 = Test3;
            Func<string, int, bool> f6 = Test3;

            bool? res;
            res = f1?.Invoke();
            res = f2?.Invoke();
            res = f3?.Invoke("");
            res = f4?.Invoke("");
            res = f5?.Invoke("", 0);
            res = f6?.Invoke("", 0);

            bool Test1()
            {
                Console.WriteLine("I Test1");
                return false;
            }

            bool Test2(string t)
            {
                Console.WriteLine("I Test2");
                return false;
            }

            bool Test3(string t, int x)
            {
                Console.WriteLine("I Test3");
                return false;
            }

            if (System.Diagnostics.Debugger.IsAttached)
            {
                Console.Write("Press any key to continue . . . ");
                Console.ReadKey();
            }
        }
    }

    Predicate

    Video Predicate (3 min.)

    Se video

    Predicate dækker en metode med et argument der altid returnerer en bool, og typen på argumentet angives generisk. Eksempelvis benyttes en Predicate i mange Array/LINQ metoder - herunder findes det index hvor værdien = 6. FindIndex-metoden tager en Predicate (delegate som returnerer en bool med en int som argument) for at findes det konkrete indeks.

    int[] array = { 5, 1, 4, 1, 6, 7, 10 };
int findes = System.Array.FindIndex(array, FindesTal);  // 4

bool FindesTal(int a)
{
    return a == 6;
}

    Så følgende tre delegates er erstattet af Predicate-delegates:

    using System;

namespace Demo
{

    public delegate bool MinDelegate2(string txt);

    internal class Program

            MinDelegate1 f1 = Test1;
            Predicate<int> f2 = Test1;

            MinDelegate2 f3 = Test2;
            Predicate<string> f4 = Test2;

            MinDelegate3 f5 = Test3;
            Predicate<double> f6 = Test3;

            bool? res;
            res = f1?.Invoke(0);
            res = f2?.Invoke(0);
            res = f3?.Invoke("");
            res = f4?.Invoke("");
            res = f5?.Invoke(0);
            res = f6?.Invoke(0);

            bool Test1(int a)
            {
                Console.WriteLine("I Test1");
                return false;
            }

            bool Test2(string t)
            {
                Console.WriteLine("I Test2");
                return false;
            }

            bool Test3(double a)

                Console.WriteLine("I Test3");
                return false;
            }

    Lambda expressions

    Video Lambda expressions (13 min.)

    Se video

    Lambda i C# er i virkeligheden ret kompleks og dækker over expressions trees og generering af dynamiske udtryk - men i sin simple form er det bare en anden og hurtigere måde at skrive en metode (sukker metode syntaks). Man kan også kalde det en anonym metode fordi en Lambda returnerer en delegate, og ikke i sig selv har et navn.

    Man kan skrive Lambda-udtryk på mange måder hvilket gør det lidt svært at lære, men når du først har styr på det er det en super effektiv måde at skrive kode på. Her er eksemplet fra tidligere med FindIndex - nu med lambda:

    int[] array = { 5, 1, 4, 1, 6, 7, 10 };
int findes1 = System.Array.FindIndex(array, FindesTal);  // 4
bool FindesTal(int a)
{
    return a == 6;
}

// var noget nemmere at skrive (hvis man lige vender sig til syntaksen - smiler)
int findes2 = System.Array.FindIndex(array, i => i == 6);   // 4

    Her er nogle af reglerne som lambda-syntaks

    • => betyder “sendes ind i metoden som argument”
    • Når der angives argumenter
    • Du behøver ikke at angive typenavn
    • Ved et argument behøver man ikke angive parentes
    • Ved flere argumenter skal der benyttes parentes
    • Ingen argumenter angives som ()
    • Hvis metoden blot består af en enkelt instruktion behøver man ikke tuborgklammer
    • Hvis metoden blot returnerer en værdi i første instruktion behøver man ikke return
    using System;

namespace Demo
{
    internal class Program
    {
        private static void Main(string[] args)
        {

            void Test1()
            {
                Console.WriteLine("I Test1");
            }
            Action f1 = Test1;
            Action f2 = () => { Console.WriteLine("I Test1"); };
            Action f3 = () => Console.WriteLine("I Test1");
            f1();
            f2();
            f3();


            void Test2(int a)
            {
                Console.WriteLine($"I Test2 med {a}");
            }
            Action<int> f4 = Test2;
            Action<int> f5 = (int x) => { Console.WriteLine($"I Test2 med {x}"); };
            Action<int> f6 = (x) => { Console.WriteLine($"I Test2 med {x}"); };
            Action<int> f7 = x => Console.WriteLine($"I Test2 med {x}");
            f4(1);
            f5(1);
            f6(1);
            f7(1);

            void Test3(int a, int b)
            {
                Console.WriteLine($"I Test2 med {a} og {b}");
            }
            Action<int, int> f8 = Test3;
            Action<int, int> f9 = (int x, int y) => { Console.WriteLine($"I Test2 med {x} og {y}"); };
            Action<int, int> f10 = (x, y) => { Console.WriteLine($"I Test2 med {x} og {y}"); };
            Action<int, int> f11 = (x, y) => Console.WriteLine($"I Test2 med {x} og {y}");
            f8(1, 1);
            f9(1, 1);
            f10(1, 1);
            f11(1, 1);

            int Test4(int a)
            {
                return a + 1;
            }
            Func<int, int> f12 = Test4;
            Func<int, int> f13 = (int x) => { return x + 1; };
            Func<int, int> f14 = (x) => { return x + 1; };
            Func<int, int> f15 = x => x + 1;
            Console.WriteLine(f12(1));
            Console.WriteLine(f13(1));
            Console.WriteLine(f14(1));
            Console.WriteLine(f15(1));

            int Test5(int a, int b)
            {
                return a + b;
            }
            Func<int, int, int> f16 = Test5;
            Func<int, int, int> f17 = (int x, int y) => { return x + y; };
            Func<int, int, int> f18 = (x, y) => { return x + y; };
            Func<int, int, int> f19 = (x, y) => x + y;
            Console.WriteLine(f16(1, 1));
            Console.WriteLine(f17(1, 1));
            Console.WriteLine(f18(1, 1));
            Console.WriteLine(f19(1, 1));

            bool Test6(int a)
            {
                return a < 10;
            }
            Predicate<int> f20 = Test6;
            Predicate<int> f21 = (int x) => { return x < 10; };
            Predicate<int> f22 = (x) => { return x < 10; };
            Predicate<int> f23 = x => x < 10;
            Console.WriteLine(f20(1));
            Console.WriteLine(f21(1));
            Console.WriteLine(f22(1));
            Console.WriteLine(f23(1));

            if (System.Diagnostics.Debugger.IsAttached)
            {
                Console.Write("Press any key to continue . . . ");
                Console.ReadKey();
            }
        }
    }
}

    Discards

    Fra C# 9 er det muligt at benytte descards (_) til at angive både kompiler og udvikler at der ikke er behov for et argument. Det bruges især i tuples, ved ignorering af returværdier fra en metode men også nogle gange i lambda:

    Func<int, int, int> a = (_, _) => 0;

    Brug af var med lambda

    Fra C# 10 er det muligt at lade kompileren finde returtypen. Alternativt kan den som noget nyt angives ved deklaration:

    Func<int, int, int> f1 = (a, b) => a + b;
var f2 = (int a, int b) => a + b;
var f3 = int (int a, int b) => a + b;

    Anden brug af lambda syntaks

    Til orientering kan lambda syntaks også benyttes til egenskaber og metoder i typer - eksempelvis

    using System;

namespace demo
{

    internal class Program
    {
        private static void Main(string[] args)
        {

            Person p = new Person { Alder = 30 };
            Console.WriteLine(p.BeregnetFødselsår);
            p.Skriv();
            Console.WriteLine(p.ErÆldre(31));
        }
    }

    public class Person {

        // Egenskaber (get/set)
        private int alder;
        public int Alder
        {
            get => alder;
            set => alder = value;
        }

        // Egenskab (get)
        public int BeregnetFødselsår => DateTime.Now.Year - Alder;

        // Metoder
        public void Skriv() => Console.WriteLine("Person er fra " + BeregnetFødselsår);
        public bool ErÆldre(int andenAlder) => alder > andenAlder;

    }
}
