    Git og GitHub

    Om modulet

    I dette modul bliver du introduceret til Git og GitHub, samt brugen af begge i Visual Studio.

    Fra WikiPedia

    Git

    Git er et frit, distribueret versionsstyringssystem med fokus på hastighed, designet til at vedligeholde kildekode. Git blev skabt af Linus Torvalds til udvikling af Linuxkernen.

    Alt er som udgangspunkt gemt lokalt, og baseret på en konkret rod-mappe hvor alt automatisk er under versionsstyring. Man kan dog vælge hvilke filer og mapper man ønsker udtaget af versionsstyring gennem en .gitignore fil i roden af mappen. Se github/gitignore, og evt. en fil direkte relateret til C# løsninger.

    GitHub

    GitHub er en af de mange cloududbydere af versionstyring baseret på Git. GitHub blev købt af Microsoft i 2018 for en ufattelig masse penge. Det er gratis at oprette både offentlige og (et bekrænset antal) private repositories, men et abonnement kan købes for få \$.

    Git, GitHub og Visual Studio

    Git er indbygget i de sidste nye versioner af Visual Studio, og de fleste installationer sørger ligeledes for, at GitHub Extension for Visual Studio bliver installeret. Ellers bør du selv gøre det.

    Tilføj VS løsning til source control

    Som udgangspunkt er en ny løsning ikke under versionsstyring så det skal du selv gøre.

    • Højreklik på løsningen i Solution Explorer-vinduet, og vælg “Add solution to source control”.

    Gem rettelser (lokalt)

    Når du ønsker at gemme rettelser kan du gøre det i Team Explorer-vinduet (typisk placeret i samme vindue som Solution Explorer).

    • Find Team Explorer-vinduet
    • Klik på huset i toppen af vinduet
    • Klik på Changes-knappen
    • Skriv en tekst (commit message)
    • Klik på Commit all-knappen

    Ny er rettelser gemt som et commit, og kan ses under Sync (huset i toppen af vinduet og Sync-knappen). Bemærk, at der kun er gemt lokalt - der er ikke synkroniseret med en Git-udbyder (som eksempelvis GitHub).

    Opret repository i GitHub

    Når først et projekt er under Git source control kan du vælge at synkronisere din kode med en cloud udbyder at Git. Der er mange at vælge i mellem, men det nemmeste er klar at benytte GitHub da det er indbygget i VS (GitHub er ejet af Microsoft).

    Først skal du oprette en (gratis) konto hos GitHub, og sørge for at din mailadresse er valideret inden du går videre. Herefter skal du logge ind i GitHub fra Visual Studio via Team Explorer-vinduets Sync-fane (se evt video). Herefter kan du oprette projektet i GitHub.

    Synkronisér med GitHub

    Når du har gemt rettelser kan du finde dem under Sync (huset i toppen af vinduet og Sync-knappen), og det er også her du skal synkronisere med eksempelvis GitHub ved at klikke på Sync-linket i toppen af vinduet. Du kan også klikke på Push-linket men bemærk, at eventuelle rettelser sket fra andre maskiner så ikke synkroniseres med dit lokale repository.

    Åbn et nyt projekt direkte fra GitHub

    Hvis du gerne vil oprette projekt baseret på et GutHub-projekt kan dette også gøres gennem Visual Studio. I det første opstartsvindue, eller fra File-menuen i VS, kan du vælge “Clone or check out code”. Her kan du angive en git-url (den kan du finde under den store grønne knap på et repository i GitHub) og en lokal placering. Så vil VS clone koden og åbne projektet. Husk at sikre dig, at Solution Explorer-vinduet er i “Solution view” (og ikke i “Folder view”). Se eventuelt video.
