    Nedarvning

    Om modulet

    Nedarvning er grundleddet i objektorientering programmering! Det giver en meget effektiv mulighed for genbrug af kode, samt mulighed at arbejde objekter i samme arvehierarki på en helt speciel måde. Sidstnævnte kaldes polymofi, og det venter vi med at se på til næste modul. I dette fokuserer vi på nedarvning - som i sig selv faktisk er ret simpelt at forstå.

    Overordnet om arv

    Nedarvning er et af principperne i objektorienteret programmering

    I sin helt grundlæggende form kan nedarvning bruges til at opbygge et hierarki hvor medlemmer (felter, metoder, egenskaber, hændelser) i klasser kan genbruges automatisk og uden kode i andre klasser som arver fra en anden klasse.

    • “Er en” relation
    • Generalisering -> Specialisering
    • Kan alt hvad mor kan + (evt) yderligere medlemmer
    • Single/Multible nedarvning
    • Kun en mor i C#

    Syntaks for nedarvning

    Man benytter kolon (:) til nedarvning

    // Mor-Barn
class Mor { }
class Barn : Mor { }

// Dyr
class Dyr { }
class Hvirveldyr : Dyr { }
class Hvirvelløse : Dyr { }
class Fisk : Hvirveldyr { }
class Torsk : Fisk { }
class Pattedyr : Hvirveldyr { }
class Elefant : Pattedyr { }
class Bløddyr : Hvirvelløse { }
class Musling : Bløddyr { }

// Personer
class Person { }
class Medarbejder : Person { }
class Kursist : Person { }
class Instruktør : Medarbejder { }
class FremmødeKursist : Kursist { }
class OnlineKursist : Kursist { }
class HjælpeInstruktør : Instruktør { }

    Tilgang til medlemmer

    Der er fortsat de to klassiske virkefelter

    • public (alle udefra og i hierarki kan tilgå medlem)
    • private (kun klassen selv kan tilgå medlem)

    men der tilføjes en ny

    • protected (alle klasser i hierarki kan tilgå medlem - men ikke udefra)
    using System;

namespace Arv1
{

    public class Mor
    {
        public int A { get; set; }
        protected int B { get; set; }
        private int C { get; set; }

        public void Test1()
        {
            // Alle egenskaber kan ses
            Console.WriteLine("I Test1");
        }

        protected void Test2()
        {
            // Alle egenskaber kan ses
            Console.WriteLine("I Test2");
        }

        private void Test3()
        {
            // Alle egenskaber kan ses
            Console.WriteLine("I Test3");
        }
    }

    public class Barn : Mor
    {
        public int D { get; set; }
        public void Test4()
        {
            // A, B og D kan ses
            // Test1, Test2 kan ses            
            Console.WriteLine("I Test4");
        }
    }

}

    Prøv denne kode:

    Arv1.Mor a = new Arv1.Mor();
a.A = 1;            
// a.B kan ikke ses
// a.C kan ikke ses
a.Test1();
// a.Test2(); kan ikke ses
// a.Test3(); kan ikke ses

Arv1.Barn b = new Arv1.Barn();
b.A = 1;
// a.B kan ikke ses
// a.C kan ikke ses
b.D = 1;
b.Test1();
// a.Test2(); kan ikke ses
// a.Test3(); kan ikke ses
b.Test4();

/*
 ---------- Output: ----------

I Test1
I Test1
I Test4

*/

    Default constructor

    • Alle default constructors bliver afviklet
    • Først mor så barn
    using System;
using System.Collections.Generic;
using System.Text;

namespace Arv2
{
    class A
    {
        public A()
        {
            Console.WriteLine("I A");
        }
    }

    class B : A {
        public B()
        {
            Console.WriteLine("I B");
        }
    }
}

    Arv2.B b = new Arv2.B();


/*
 ---------- Output: ----------

I A
I B

*/

    Custom constructor

    Custom constructors nedarves ikke

    • Dem skal man selv tilføje (hvis de ønskes - hvilket jo netop er årsagen til at de ikke nedarves)
    using System;
using System.Collections.Generic;
using System.Text;

namespace Arv2a
{
    class A
    {
        public A()
        {
            Console.WriteLine("I A");
        }

        public A(int x) {
            Console.WriteLine("I A(x)");
        }
    }

    class B : A {
        public B()
        {
            Console.WriteLine("I B");
        }
    }
}

    Arv2a.B b = new Arv2a.B();

// Custom arves ikke
// Arv2a.B b = new Arv2a.B(3);


/*
 ---------- Output: ----------

I A
I B

*/

    Hvis man tilføjer en custom constructor til en nedarvet klasse kaldes først default constructor frs “mor” og herefter custom constructor’en.

    base

    • Brug base til få fat i øvre constructors
    • Lidt som this - bare for arv
    • Husk: custom constructor arves ikke
    using System;
using System.Collections.Generic;
using System.Text;

namespace Arv3
{
    public class A
    {
        public A()
        {
            Console.WriteLine("Default i A");
        }
        public A(int a)
        {
            Console.WriteLine("Custom i A");
        }
        public A(int a, int b)
        {
            Console.WriteLine("Custom speciel i A");
        }
    }

    public class B : A
    {
        public B()
        {
            Console.WriteLine("Default i B");
        }
        public B(int v) : base(v)
        {
            Console.WriteLine("Custom i B");
        }
    }

    public class C : B
    {
        public C()
        {
            Console.WriteLine("Default i C");
        }
        public C(int c) : base(c)
        {
            Console.WriteLine("Custom i C");
        }
    }
}

    Arv3.C c = new Arv3.C(1);

/*
 ---------- Output: ----------

Custom i A
Custom i B
Custom i C

*/
